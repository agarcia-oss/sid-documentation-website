Common Domain::Trouble or Problem ABE
Class Problem

General description of a problem that can be shared between service and customer.
A Problem is an indication that an entity is no longer functioning according to the agreement with its client(s).

Attributes
String  problemId

Identifier of the problem. Note that this identifier is local to the server side of the interface.
Only EntityIdentifier can be considered as global.
String  originatingSytem

Indicates where the problem was generated.
Integer  impactImportanceFactor
Integer  priority

An indication varying from 1 to 10 of how important it is for the service provider to correct the Service Problem. The priority can be changed by the human operator while the impact importance can only be changed by the system which determines the impact.

String  description

Free form text describing the Service Problem.
String  firstAlert
String  category

Classifier for the problem. Settable. Structured text/ enum.
In the ATIS Service Outage document, the What Category
corresponds to this attribute and the values can be used as possible values.
String  responsibleParty

Person or organization responsible for handling this problem. This is text or structured text and not an association to a party object. The Who category from the ATIS Service Outage document can be used for this attribute.
Integer  problemEscalation

Indicates if this service problem has been escalated or not. Possible values are 0 to 10. A value of zero means no escalation.
The meanings of values 1-10 are to be determined by the user of the interface, but they show increasing levels of escalation.

String  comments

Comments on problem, as a list of comments.
DateTime  timeRaised

Time the problem was raised
DateTime  timeChanged

Time the problem was last changed
String  reason

Free text or optionally structured text. It can be Unknown. The Why category of the ATIS Service Outage document can be used to fill this value.
