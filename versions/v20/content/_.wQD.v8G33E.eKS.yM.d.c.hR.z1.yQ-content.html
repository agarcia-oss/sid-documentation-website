Diagram Figure ResCS.01 - Software Resource Specification
The following Figure shows the details of the corresponding Software Specification classes.
SoftwareResourceSpecification in the figure depicts the attributes that are shared by every instance of an associated SoftwareResource entity.
The SoftwareSpecification entity has been shown separately from the APISpecification entity because it contains attributes unique to it as shown in the figure. For an example of APISpecification entity usage please refer to Root Guidebook.
The composite/atomic pattern has been applied to the SoftwareSpecification entity to allow for groupings of Software, such as the software module specifications that are contained within an application specification. Note that the application specification is not shown as a subclass of SoftwareSpecification. This is because there have been no unique attributes discovered for it.
Note that there are many attributes and associations inherited by SoftwareResourceSpecification, such as the association to SupplierPartner from ResourceSpecification as well as others from LogicalResourceSpecification (Refer to the corresponding section).
