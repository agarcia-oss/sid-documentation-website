Resource Domain::Resource Trouble ABE::Alarm Severity Assignment Profile ABE
Data Type AlarmSeverityAssignment

This data type provides three values for severity to cover the three cases of service affecting conditions, non- service affecting conditions and conditions where the service impact is unknown. The structure identifies the specific case of alarm that it applies to using three probable cause identifiers in combination.

See also attached supporting document for further details on the values of the strings probableCause, probableCauseQualifier and nativeProbableCause.

Attributes
String  nativeProbableCause

This attribute identifies the probable cause used on the NE/target OS. This may also be used to qualify the probable cause to achieve uniqueness in some cases where the probable cause is not sufficient.
AssignedSeverity  nonServiceAffectingSeverity

This attribute identifies the severity assigned to the probable cause when not service affecting.
String  probableCause

This attribute identifies the probable cause of the alarm to which the severities should be applied to.
See attached supporting document SD1-33_ProbableCauses.
AssignedSeverity  serviceAffectingSeverity

This attribute identifies the severity assigned to the probable cause when service affecting.
AssignedSeverity  serviceIndependentSeverity

This attribute identifies the severity assigned to the probable cause when the reportable alarm is service independent or the service impact is not known.
String  specificProblem

This attribute identifies a qualifier of the probable cause that is used to achieve uniqueness in some cases where the probable cause is not sufficient.
