The development of the model for the portability uses all the ABEs previously described and how each one of them relates to the other. There is an inheritance for the main entity NumberPortabililityRequest to see the subdivision more clearly. This entity has four types of requests for the number portability, NumberPortInRequest, NumberPortOutRequest, NumberPortOverRequest and NumberRetirementRequest. This main concrete class, NumberPortabilityRequest, has an aggregation for all the portability roles applied during the process. NumberPortabilityRole inherits all the behavior and attributes of LogicalResourceRole and relates to existing and new parties.
NumberPortabilityRequest also forms a relation of many-to-many with the rules, with an association class in the middle called PortabilityRequestPolicyRule for the information in common.
