The diagram below lists the Resource Domain's ABEs Level 1 & 2.

[] Physical Resource Specification ABE
• The Physical Resource Specification ABE describes the invariant characteristics and behavior (attributes, methods, constraints, and relationships) of a PhysicalResource.
[] Logical Resource Specification ABE
• The Logical Resource Specification ABE is used to define the invariant characteristics and behavior (attributes, methods, constraints, and relationships) of a LogicalResource.
[] CompoundResource Specification ABE
• The CompoundResource Specification ABE used to define the invariant characteristics and behavior (attributes, methods, constraints, and relationships) of a CompoundResource.
[] Resource Catalog ABE
• A collection of entities that represent the specification of ResourceCatalogs and catalogs described by them that contain ResourceSpecifications.
[] Physical Resource ABE
• The Physical Resource ABE contains entities that are used to represent the various aspects of a Physical Resource.
[] Logical Resource ABE
• The Logical Resource ABE contains entities that are used to represent the various aspects of a LogicalResource.
[] Compound Resource ABE
• The Compound Resource ABE contains entities that are used to represent the various aspects of a CompoundResource.
CompoundResource is a set of entities that shows how to aggregate PhysicalResources and LogicalResources into aggregate entities that has physical and/or logical characteristics and behavior.
[] ResourceRole System View and TIP Resource ABEs
• not yet described
[] Resource Number Portability ABE
• The Resource Number Portability ABE contains entities grouped for the portability itself. This means all the processes involved, like Port In, Port Out, Port Over, and Retirement of numbers.
[] Resource Number Portability Role ABE
• The Resource Number Portability Role ABE contains entities related to the roles taked for the portability.
[] Alarm & Alarm Severity Assignment Profile ABEs not yet described
