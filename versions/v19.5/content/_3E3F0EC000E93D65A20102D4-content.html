Diagram Figure Proj.05 - specification
Following Figure also shows a new Entity, ProjectResource. For now, we will use a simple definition for ProjectResource, as being an input to an Activity that can be consumed, modified or utilized by the Activity.
e.g. If we have an Activity “Weld Chair” then we may have ProjectResources such as chair parts, welding rod, a welder tradesman, a welding machine and electricity.
e.g. If we have an Activity “Upgrade Router” then we may have ProjectResources of firmware CD, technician, upgraded IO card, Laptop PC.
Note that in reality, we don’t constrain ProjectResources only to Activity inputs but also allow other things that add to the ProjectCost to be included (maybe the cost of capital, if a Project involves borrowing money!).
