In some cases there are limits to the number of ProductOfferings that can be procured as part of a BundledProductOffering. For example, when subscribing to a mobile telephone service up to five phones can be procured at a special price. The BundledProdOfferOption handle these cases by allowing lower and upper limits to be set for ProductOfferings that comprise a BundledProductOffering as shown in the following figure.
When a ProductOffering is bundled within another ProductOffering, the question arises as to whether the ProductOffering can be sold stand-alone, or only within a containing bundle.
To describe this, two options exist:
• Option 1: use the isSellableStandAlone attribute to specify it
o In this case pricing rules will be more complex but only one ProductOffering will be seen at the catalog level
• Option 2: create specific ProductOffering for the same ProductSpecification that can be sold stand-alone.
o In this case princing rules will be simpler but many ProductOfferings for the same ProductSpec will be visible at the catalog level
If the isSellableStandAlone attribute is TRUE, it indicates that the ProductOffering, which is part of a BundledProductOffering, can also be offered for sale stand-alone, if FALSE means that the ProductOffering can only be sold within a BundledProductOffering. NOTE: This attribute only applies to offerings that are part of a BundledProductOffering.
